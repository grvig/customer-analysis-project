Business Meaning

call_type represents the type of customer interaction.

Valid call_type values:

* Complaint
* Service Inquiry
* General Query
* Follow Up

Complaint = actual customer complaint.

Service Inquiry = customer asking about service-related matters.

General Query = informational request.

Follow Up = status update or previous issue follow-up.

issue_category represents the topic discussed during the interaction.

service_cost represents revenue generated by a service.

customer_rating represents customer satisfaction from 1 to 5.

vehicle make and model are stored in vehicles.make and vehicles.model.

priority represents complaint severity.

Valid priority values:

* High
* Medium
* Low

When filtering by priority, always use:

priority = 'High'
priority = 'Medium'
priority = 'Low'

Never use:

priority = 1
priority = 2
priority = 3

Examples

Question:
How many customers are in the database?

SQL:
SELECT COUNT(*) FROM customers;

Question:
Show the top 5 complaint categories.

SQL:
SELECT
issue_category,
COUNT(*)
FROM calls
WHERE call_type = 'Complaint'
GROUP BY issue_category
ORDER BY COUNT(*) DESC
LIMIT 5;

Question:
What are the most common complaints for TVS Jupiter vehicles?

SQL:
SELECT
issue_category,
COUNT(*) AS complaint_count
FROM calls
JOIN vehicles
ON calls.vehicle_id = vehicles.vehicle_id
WHERE
call_type = 'Complaint'
AND make = 'TVS'
AND model = 'Jupiter'
GROUP BY issue_category
ORDER BY complaint_count DESC;

Question:
Which agents handled the most high priority complaints?

SQL:
SELECT
a.agent_name,
COUNT(*) AS complaint_count
FROM calls c
JOIN agents a
ON c.agent_id = a.agent_id
WHERE
c.call_type = 'Complaint'
AND c.priority = 'High'
GROUP BY a.agent_name
ORDER BY complaint_count DESC;

Question:
Show battery complaints from Anna Nagar.

SQL:
SELECT
issue_category,
COUNT(*) AS complaint_count
FROM calls
WHERE
call_type = 'Complaint'
AND branch = 'Anna Nagar'
AND issue_category ILIKE '%Battery%'
GROUP BY issue_category
ORDER BY complaint_count DESC;

Question:
Which branch generated the highest service revenue?

SQL:
SELECT
c.branch,
SUM(s.service_cost) AS revenue
FROM services s
JOIN calls c
ON s.call_id = c.call_id
GROUP BY c.branch
ORDER BY revenue DESC
LIMIT 1;

Important Join Patterns

Revenue By Branch

services does NOT contain branch.

To calculate revenue by branch:

services
JOIN calls
ON services.call_id = calls.call_id

Use:

GROUP BY calls.branch

Customer Rating By Branch

surveys does NOT contain branch.

To calculate ratings by branch:

surveys
JOIN services
ON surveys.service_id = services.service_id

ON services.call_id = calls.call_id

Use:

Average Calculations

When calculating averages for analytics, dashboards, or reports,
prefer:

ROUND(AVG(column_name), 2)

instead of:

AVG(column_name)

unless the user explicitly requests full precision.

Aggregate Filtering Rules:

- Use WHERE for filtering individual rows.
- Use HAVING for filtering aggregate values.
- Examples:

Average rating above 3.5:
HAVING AVG(customer_rating) > 3.5

Revenue above 100000:
HAVING SUM(service_cost) > 100000

More than 50 complaints:
HAVING COUNT(*) > 50

Complex analytics queries should prefer WITH clauses (CTEs).

Example:

WITH branch_stats AS (
    SELECT
        branch,
        SUM(service_cost) AS revenue,
        AVG(customer_rating) AS avg_rating
    FROM ...
    GROUP BY branch
)

SELECT *
FROM branch_stats
WHERE revenue > (
    SELECT AVG(revenue)
    FROM branch_stats
);
